DynEval

A Concordion extension that dynamically evaluates markups in the specification documents.

What DynEval does

Evaluating marked-up references

DynEval evaluates references marked between \${ and \}, and replaces the marked references with a matching value, which it takes from a configured values provider.
In the specification document, this would be written as: <b concordion:set="#text">\${simpleReference\}</b>

Given an adequately-prepared values' provider which matches the reference \${simpleReference\} to the value "Hello, simple reference!",
When setting a Concordion parameter to "\${simpleReference\}"
Then the parameter will actually be parsed by DynEval to "${simpleReference}" in the specification document
And a "display()" method call with that parameter should return "Hello, simple reference!".

One is not limited in using references with a concordion:set attribute. References can be used anywhere in a document!

She said "${simpleReference}".
"${hi}", he replied.

One could also use references to specify the expected result of an assertEquals.

Given an adequately-prepared values' provider,
When calling the "display()" fixture method with the Concordion parameter "Hi there!"
Then it is possible to assert the method's result against a "\${hi\}" reference that holds the result to be obtained: "${hi}"

It is possible to use multiple references, for example to set up a Concordion parameter. The references will then be concatenated:

Given an adequately-prepared values' provider,
When setting a Concordion parameter to "\${one\}\${two\}\${three\}"
Then the parameter will be evaluated to "${one}${two}${three}" and displayed by the "display()" as "123".

Escaping its own markups

Normally DynEval parses the whole specification document in search of markups to evaluate.
It is however possible to specifically escape the parsing of a marked expression.

When setting a Concordion parameter to "\\${escapedMarkup\\}", which indicates that "\${escapedMarkup\}" should not be evaluated (that is, replaced with a value)
Then Concordion should not evaluate the marked expression, and a "display()" method call with that parameter should return "${escapedMarkup}".
